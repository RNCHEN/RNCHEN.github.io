对新手很友好的搭建博客和使用,包含图床配置(因为我本身就是新手)

使用hexo+github配置博客

并使用picgo+gitee配置图床使得上传博客变得轻松

按照网上的教程来,但是基本每一步都有问题 ：（

所以这篇文章还是算对大家伙比较友好且完整的拉 ：）

使用typora时的配置图床

配置图床的必要性,平时写typora的时候图片是插入本地的,及其不方便文档的传阅,也不方便博客的上传.

配置图床后只要有网络,那么图片的传阅就很方便啦.

版本要求

  • PicGo 2.2.0 及以上。
  • typora 0.9.84版本及以上

picgo下载

https://github.com/Molunerfinn/PicGo(可能跳转有点慢)

向下滑

windows用户选择这个黄色圈圈,mac用户详情见说明

Gitee的配置

官网如下:

https://gitee.com/

第一步新建仓库

名称任意,路径可不填,但是记得选上公开

选上Readme并点击下方创建即可

回到右上角,点击设置

点击左边一栏选择私人令牌(注意这里和GitHub上的)

只选择projects一项

建议复制后选择一个地方存储令牌(其实后面也只用得到一次)

配置图床

打开picgo,左侧最下方插件设置中选择

随后在图床设置中选择gitee

点击确定并设置为默认图床

验证方式:拖动图片查看是否上传成功,可在相册中查看

Typora中的更改

打开typora->file(左上角)->preference->image

PicGo Path选择最开始picgo下载.exe文件位置

然后Test UpLoader

==注意==如果出现

则检查Test窗口出现的

与picgo->picgo设置->设置server中的端口号是否一致,不一致则更改server为36677

最后如图所示则图床配置成功

这个很重要必须写

git不行的话

就得那俩变量的地址添加到path中

Error: ENOENT: no such file or directory

If you get an error like Error: ENOENT: no such file or directory it’s probably due to to mixing uppercase and lowercase letters in your tags, categories, or filenames. Git cannot automatically merge this change so it breaks the automatic branching.
